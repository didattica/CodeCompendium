## 16.4.1 Cisco AutoSecure

### Concetto

Quando un nuovo sistema operativo viene installato su un dispositivo, le impostazioni di sicurezza vengono configurate sui **valori predefiniti**. Nella maggior parte dei casi, questo livello di sicurezza è **inadeguato**.

I dispositivi appena usciti dalla fabbrica — o rimasti a lungo in magazzino — spesso hanno:
- Servizi inutili attivi per default
- Password predefinite note a tutti gli attaccanti
- Patch di sicurezza non aggiornate

**Cisco AutoSecure** è una funzione che aiuta a blindare rapidamente un router eseguendo il comando `auto secure`. È un punto di partenza utile, ma come avvisa esplicitamente il sistema stesso:

```
*** AutoSecure configuration enhances the security of
the router but it will not make router absolutely secure
from all security attacks ***
```

> Non esiste la sicurezza assoluta. AutoSecure è il primo passo, non l'ultimo.

### Le 4 Regole d'Oro da applicare subito

| Regola | Azione | Perché |
|---|---|---|
| **Cambia tutto** | Modifica subito nomi utente e password di default | Le credenziali di fabbrica sono pubblicamente note |
| **Limita gli accessi** | Solo gli utenti autorizzati accedono alle risorse | Principio del minimo privilegio |
| **Meno è meglio** | Disattiva e disinstalla servizi e applicazioni non necessari | Ogni servizio attivo è una superficie d'attacco potenziale |
| **Aggiorna subito** | Installa le patch di sicurezza prima del deployment | I dispositivi in magazzino possono avere software obsoleto con vulnerabilità note |

## 16.4.2 Password

### Concetto

La password è la **prima linea di difesa** di un dispositivo. Una password debole può essere violata in pochi secondi con strumenti automatici di brute force. Le linee guida Cisco per una password efficace sono:

| Criterio | Regola |
|---|---|
| **Lunghezza** | Minimo 8 caratteri, preferibilmente 10 o più. Più è lunga, più è sicura. |
| **Complessità** | Mescola MAIUSCOLE, minuscole, num3ri e simboli ($%&!) |
| **Cosa evitare** | Parole del dizionario, date di nascita, nomi di persone o animali, sequenze ovvie (123456, abcdef) |
| **Ortografia creativa** | Usa errori ortografici intenzionali: `Smith → 5mYth`, `Security → 5ecur1ty` |
| **Cambiamento frequente** | Aggiorna periodicamente le password — se una è compromessa, limita la finestra di rischio |
| **Non scriverle** | Mai su post-it, su carta o sul monitor |

### Passphrase — il metodo più efficace

Sui router Cisco, gli spazi iniziali nelle password vengono ignorati, ma gli spazi **dopo il primo carattere** sono validi. Questo permette di creare **passphrase**: frasi complete usate come password.

> **Esempio:** `Il Mio Gatto Mangia Pesce!`
> è più sicura di `Gx7!9p` — è più lunga, più difficile da indovinare con brute force, e molto più facile da ricordare.

### 🔑 Metafora della Serratura

> "Una password è come la chiave di una serratura. Una password di 4 caratteri è una chiave con 4 dentini semplici: facile da duplicare o forzare con un grimaldello. Una passphrase complessa è come una chiave di alta sicurezza con incisioni laser, codici magnetici e profilo irregolare — quasi impossibile da copiare."

> **💡 Domanda per la classe:** "Qual è la password del vostro smartphone? Quanti caratteri ha? Sapreste indovinare quella del compagno di banco solo guardando le sue abitudini?"

## 16.4.3 Sicurezza Password Aggiuntiva

### Concetto

Avere una password complessa non basta se viene **esposta inavvertitamente** o se un attaccante può fare tentativi illimitati per indovinarla. Cisco IOS offre quattro comandi fondamentali per rafforzare la protezione:

---

### A — Cifrare tutte le password in chiaro

```
R1(config)# service password-encryption
```

**Cosa fa:** Cifra tutte le password memorizzate in chiaro nel file di configurazione del router. Senza questo comando, chiunque riesca a vedere il `running-config` legge le password in chiaro. Con il comando attivo, al posto di `cisco` appare `03095A0F034F`.

> ⚠️ Non è una cifratura "forte" (usa un algoritmo reversibile), ma impedisce la lettura diretta a colpo d'occhio — un primo livello di protezione importante.

---

### B — Imporre la lunghezza minima delle password

```
R1(config)# security passwords min-length 8
```

**Cosa fa:** Il router rifiuta qualsiasi nuova password più corta di 8 caratteri. Impedisce configurazioni negligenti da parte di amministratori frettolosi.

---

### C — Bloccare gli attacchi Brute Force

```
R1(config)# login block-for 120 attempts 3 within 60
```

**Cosa fa:** Se vengono rilevati 3 tentativi di accesso falliti entro 60 secondi, il router blocca tutti i tentativi di login per 120 secondi.

> **💡 Perché è importante:** Un software di brute force può provare migliaia di password al secondo. Senza questo blocco, un attaccante può testare un intero dizionario di password senza mai essere fermato.

### 🚪 Metafora della Porta Blindata con Allarme

> "Immaginate una porta blindata che, dopo tre tentativi sbagliati con la chiave, attiva una sirena e si blocca per due minuti. Il ladro non può più fare tentativi rapidi — deve aspettare, e nel frattempo voi potete accorgervi dell'intrusione."

---

### D — Timeout di inattività

```
R1(config)# line vty 0 4
R1(config-line)# exec-timeout 5 30
```

**Cosa fa:** Disconnette automaticamente un utente inattivo dopo **5 minuti e 30 secondi**. Per default, i router Cisco disconnettono dopo 10 minuti — questo comando riduce ulteriormente la finestra di rischio.

> **Scenario reale:** Un amministratore apre una sessione privilegiata sul router, si alza per andare a pranzo e dimentica la sessione aperta. Senza timeout, chiunque passi davanti al terminale può modificare la configurazione del dispositivo.

### Configurazione completa di esempio

```
R1(config)# service password-encryption
R1(config)# security passwords min-length 8
R1(config)# login block-for 120 attempts 3 within 60
R1(config)# line vty 0 4
R1(config-line)# password cisco123
R1(config-line)# exec-timeout 5 30
R1(config-line)# transport input ssh
R1(config-line)# end
```

Dopo aver applicato `service password-encryption`, il `show running-config` mostrerà:

```
line vty 0 4
 password 7 094F471A1A0A
 exec-timeout 5 30
 login
 transport input ssh
```

## 16.4.4 Abilitare SSH

### Concetto

Per gestire un router da remoto esistono due protocolli: **Telnet** e **SSH**. La differenza è fondamentale:

| | Telnet | SSH (Secure Shell) |
|---|---|---|
| **Crittografia** | ❌ Nessuna — tutto in chiaro | ✅ Traffico completamente cifrato |
| **Sicurezza** | ❌ Pericoloso — intercettabile | ✅ Sicuro per uso in produzione |
| **Uso consigliato** | ❌ Mai in produzione | ✅ Sempre |

### 📬 Metafora Telnet vs SSH

> "Usare Telnet per gestire un router è come scrivere la password su una cartolina postale: chiunque la maneggi lungo il percorso — il postino, l'ufficio smistamento, il vicino curioso — può leggerla.
> SSH invece è come inviare una lettera in una busta blindata con serratura: solo il destinatario ha la chiave per aprirla."

### Configurare SSH in 6 Passi

**Passo 1 — Nome host univoco**
```
Router(config)# hostname R1
```
Il dispositivo deve avere un nome univoco — è necessario per generare le chiavi SSH.

**Passo 2 — Nome di dominio IP**
```
R1(config)# ip domain-name span.com
```
Il nome dominio viene combinato con l'hostname per creare l'identità univoca del dispositivo nella generazione delle chiavi.

**Passo 3 — Generare le chiavi RSA**
```
R1(config)# crypto key generate rsa general-keys modulus 1024
```
Genera la coppia di chiavi crittografiche che SSH usa per cifrare il traffico. Il modulo determina la dimensione della chiave (da 360 a 2048 bit). **Minimo consigliato: 1024 bit.** Chiavi più grandi = più sicurezza, ma anche più tempo per cifrare/decifrare.

**Passo 4 — Creare l'utente nel database locale**
```
R1(config)# username Bob secret cisco
```
Il parametro `secret` cifra la password con MD5 — più sicuro del semplice `password`.

**Passo 5 — Autenticazione sul database locale**
```
R1(config)# line vty 0 4
R1(config-line)# login local
```
Dice alle linee vty di usare il database locale per autenticare gli utenti.

**Passo 6 — Abilitare solo SSH in ingresso**
```
R1(config-line)# transport input ssh
```
Blocca Telnet e permette **esclusivamente** connessioni SSH sulle linee vty.

### Configurazione completa

```
Router# configure terminal
Router(config)# hostname R1
R1(config)# ip domain-name span.com
R1(config)# crypto key generate rsa general-keys modulus 1024
R1(config)# username Bob secret cisco
R1(config)# line vty 0 4
R1(config-line)# login local
R1(config-line)# transport input ssh
R1(config-line)# exit
```

> **💡 Attività in classe:** Dopo la configurazione, tenta di collegarti via Telnet. Il sistema rifiuterà la connessione. Spiega agli studenti che **questo è esattamente il risultato corretto** — il router sta funzionando come dovrebbe.

## 16.4.5 Disabilitare i Servizi Non Utilizzati

### Concetto

Router e switch Cisco avviano con un elenco di **servizi attivi di default** — alcuni necessari, altri no. Ogni servizio attivo inutilmente:
- Consuma risorse (CPU e RAM)
- Rappresenta una **superficie d'attacco** potenziale per gli attori delle minacce

### Verificare le porte aperte

Su **IOS-XE** (versioni recenti):
```
Router# show ip ports all
```

Output tipico di un router IOS-XE pulito:
```
Proto  Local Address          Foreign Address    State      PID/Program Name
tcp    :::443                 :::*               LISTEN     309/[IOS]HTTP CORE
tcp    *:443                  *:*                LISTEN     309/[IOS]HTTP CORE
udp    *:67                   0.0.0.0:0                     387/[IOS]DHCPD Receive
```
Solo HTTPS (porta 443) e DHCP (porta 67) — configurazione accettabile.

Su **IOS precedente a IOS-XE** (dispositivi più vecchi):
```
Router# show control-plane host open-ports
```

Se l'output mostra Telnet (porta 23) o HTTP non sicuro (porta 80), questi vanno **disabilitati immediatamente**:

```
Router(config)# no ip http server           ← disabilita HTTP non sicuro
Router(config)# line vty 0 15
Router(config-line)# transport input ssh    ← disabilita Telnet, permette solo SSH
```

### 🪟 Metafora delle Finestre

> "Ogni servizio attivo è una finestra della vostra casa. Finché siete svegli e la sorvegliate, va bene. Ma di notte, ogni finestra aperta che non serve è un'opportunità per un ladro. La regola è semplice: se non usate quella finestra, chiudetela."

> **💡 Suggerimento didattico:** Mostrate in Packet Tracer l'output di `show ip ports all` prima e dopo la disabilitazione dei servizi. La differenza visiva è immediata e molto efficace.
