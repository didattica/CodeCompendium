## 16.1 — Minacce e Vulnerabilità

> [!NOTE]
> Minaccia => Focus: L'attaccante o l'evento.
> Brute-Force Attacks, Packet Sniffing, DoS (Denial of Service),Trojan Horse e Virus
>
> La vulnerabilità è una falla o una debolezza nel sistema,
> Debolezze software: Bug nel sistema operativo IOS.

| Categoria | Esempi |
|-----------|--------|
| **Minacce esterne** | Hacker, malware, attacchi DoS |
| **Minacce interne** | Dipendenti, accessi non autorizzati |
| **Vulnerabilità SW** | Bug, patch mancanti, misconfigurazioni |
| **Vulnerabilità HW** | Accesso fisico non controllato |

**Sicurezza fisica** → spesso trascurata, ma critica  
→ Rack chiusi, accesso badge, sorveglianza

---

## 16.2 — Attacchi alla Rete

### Tipi di malware
- **Virus** – si propaga allegandosi a file
- **Worm** – si auto-replica senza intervento utente
- **Trojan** – si maschera da software legittimo
- **Ransomware** – cifra dati e chiede riscatto
- **Spyware/Adware** – raccolta dati nascosta

### Attacchi di ricognizione
- `ping sweep`, `port scan` (es. nmap)
- Obiettivo: mappare la rete prima dell'attacco

### Attacchi di accesso
- **Brute force** – tentativi ripetuti su password
- **Man-in-the-Middle (MitM)** – intercettazione traffico
- **Phishing / Social Engineering** – ingannare l'utente

### Attacchi DoS / DDoS
- **DoS** – un solo sorgente, sovraccarica il target
- **DDoS** – botnet, sorgenti multiple
- **SYN flood** – esaurisce le connessioni TCP

---

## 16.3 — Mitigazione degli Attacchi

### Difesa in profondità *(layered security)*
> Nessuno strumento è sufficiente da solo → si usano **più livelli**

```
[Perimetro] → Firewall → IDS/IPS → [Endpoint] → Antivirus → [Dati] → Encryption
```

### Best practice
- ✅ **Backup regolari** – 3-2-1 rule (3 copie, 2 media, 1 offsite)
- ✅ **Patch & aggiornamenti** – chiudere le CVE note
- ✅ **AAA** – Authentication · Authorization · Accounting
- ✅ **Firewall** – packet filter, stateful, proxy
- ✅ **Endpoint security** – antivirus, EDR, MDM

### AAA in breve
| | Cosa fa |
|--|---------|
| **Authentication** | Chi sei? (username/password, MFA) |
| **Authorization** | Cosa puoi fare? (permessi, ACL) |
| **Accounting** | Cosa hai fatto? (log, audit) |

## 16.4 — Sicurezza dei Dispositivi Cisco

### Protezione password – comandi fondamentali

```cisco
! Imposta password enable cifrata
enable secret <password>

! Password linee console e VTY
line console 0
 password <password>
 login

line vty 0 4
 password <password>
 login

! Cifra tutte le password in chiaro
service password-encryption

! Banner di avviso (obbligatorio legalmente)
banner motd # Accesso autorizzato soltanto! #

! Lunghezza minima password
security passwords min-length 8
```

### Accesso remoto sicuro (SSH vs Telnet)
> **Telnet = NO** (traffico in chiaro) · **SSH = SÌ** (cifrato)

```cisco
! Prerequisiti SSH
ip domain-name lab.local
crypto key generate rsa modulus 2048
ip ssh version 2

! Abilita SSH sulle VTY
line vty 0 4
 transport input ssh
 login local

! Utente locale
username admin secret <password>
```

### Disabilita servizi non necessari
```cisco
no ip http server          ! Disabilita HTTP
no cdp run                 ! Disabilita CDP se non necessario
no ip finger
no service tcp-small-servers
no service udp-small-servers
```

### Verifica configurazione
```cisco
show running-config
show ip ssh
show users
```

### Checklist sicurezza rapida
```
□ enable secret → configurato?
□ password console/VTY → configurata?
□ service password-encryption → attivo?
□ banner motd → presente?
□ SSH v2 → abilitato?
□ Telnet → disabilitato?
□ HTTP server → disabilitato?
□ Utente locale → creato?
```

## 🔑 Concetti chiave da ricordare

| Termine | Definizione rapida |
|---------|-------------------|
| **Vulnerability** | Debolezza sfruttabile |
| **Threat** | Potenziale causa di danno |

| **DoS** | Rende un servizio non disponibile |
| **DDoS** | DoS da sorgenti multiple (botnet) |
| **AAA** | Auth + Authz + Accounting |
| **MitM** | Intercettazione del traffico |
| **Defense in depth** | Sicurezza a più livelli |

---

### Lezione 2: Modulo 17 - Parte 1: Progettazione e Verifica (3 ore)
Iniziare il Modulo 17 qui ti permette di spiegare come si costruisce una piccola rete da zero prima di passare a risolvere i problemi.
*   **Progettazione (1,5 ore):** Analizzi i dispositivi per piccole reti (SOHO), i protocolli necessari e come scalare verso reti più grandi.
*   **Test di Connettività (1,5 ore):** Ti fermi alla sezione **17.4 (Verifica della connettività)**. È il momento perfetto per far fare pratica con i comandi `ping` e `traceroute`, spiegando bene i risultati ICMP.

---

### Lezione 3: Modulo 17 - Parte 2: Troubleshooting e Quiz (3 ore)
L'ultima lezione è la più "interattiva". Insegni ai tuoi studenti a pensare come dei sistemisti che devono riparare un guasto.
*   **Metodologia (1 ora):** Spieghi la sezione **17.6**, ovvero l'approccio sistematico al troubleshooting (le 6 fasi).
*   **Scenari Pratici (1,5 ore):** Dedichi molto tempo alla sezione **17.7**, dove gli studenti devono risolvere scenari di errore reali (es. problemi di IP, gateway o cavi).
*   **Chiusura (0,5 ore):** Svolgi il quiz finale del modulo (**17.8**) per verificare che tutti abbiano compreso.
