
# 🛠️ Guida al Laboratorio: Protezione dei Dispositivi di Rete

## 1. Parte 1: Fondamenta e Perimetro ✅
*Configurazione iniziale del dispositivo.*

*   **Cambio Nome (Hostname)** ✅:
    ```bash
    Router> enable
    Router# configure terminal
    Router(config)# hostname RTR-A
    ```
*   **Disattivazione Ricerca DNS** (Consigliato):
    ```bash
    RTR-A(config)# no ip domain-lookup
    ```
    *Metafora: Impedisce al router di "incantarsi" cercando di risolvere comandi digitati male.*

---

## 2. Parte 2: Blindare il Router (In Corso...)
*Misure di sicurezza per l'accesso e la gestione.*

### A. Sicurezza delle Password ✅
*   **Impostazione lunghezza minima (es. 10 caratteri)** ✅:
    ```bash
    RTR-A(config)# security passwords min-length 10
    ```
*   **Protezione della "Cassaforte" (Privileged Mode)** ✅:
    
```bash
    RTR-A(config)# enable secret @Sicurezza123!
    ```
*   **Protezione del "Portone" (Console)** ✅:
    ```bash
    RTR-A(config)# line console 0
    RTR-A(config-line)# password @Cons1234!
    RTR-A(config-line)# login
    RTR-A(config-line)# exit
    ```
*   **Crittografia globale di tutte le password** ✅:
    ```bash
    RTR-A(config)# service password-encryption
    ```

### B. Configurazione SSH (Accesso Remoto Sicuro) ⏳
*Da fare per permettere l'accesso via rete senza "parlare a voce alta" (Telnet).*
*   **Definire il dominio e creare l'utente**:
    ```bash
    RTR-A(config)# ip domain-name lab.sicuro.it
    RTR-A(config)# username SSHadmin secret @AdminPass123!
    ```
*   **Generare le chiavi crittografiche (serratura digitale)**:
    ```bash
    RTR-A(config)# crypto key generate rsa
    # Scegliere 1024 o superiore quando richiesto
    ```
*   **Attivare SSH sulle linee virtuali**:
    ```bash
    RTR-A(config)# line vty 0 4
    RTR-A(config-line)# transport input ssh
    RTR-A(config-line)# login local
    RTR-A(config-line)# exit
    ```

### C. Blocco per attacchi Brute Force ⏳
*   **Impostazione blocco automatico**:
    ```bash
    RTR-A(config)# login block-for 120 attempts 3 within 60
    ```
    *Metafora: Se sbagli 3 volte in un minuto, la porta rimane sbarrata per 2 minuti.*

---

## 3. Parte 3: Il Test del Ladro ⏳
*Verifica delle misure impostate.*

*   **Comando di verifica stato login**:
    ```bash
    RTR-A# show login
    ```
*   **Verifica delle password criptate**:
    ```bash
    RTR-A# show running-config
    ```

---

## 4. Parte 4: Mettere in sicurezza le "Finestre" (Switch) ⏳
*Spegnimento porte inutilizzate.*

*   **Chiusura forzata delle porte (es. dalla 10 alla 24)**:
    ```bash
    S1(config)# interface range fa0/10 - 24
    S1(config-if-range)# shutdown
    ```
    *Metafora: Murare le finestre non utilizzate per evitare intrusioni fisiche.*

---

## 5. Salvataggio Finale: Il "Sigillo" ⏳
*Fondamentale per non perdere il lavoro al riavvio.*
```bash
RTR-A# copy running-config startup-config
# Oppure il comando rapido:
RTR-A# wr
```
