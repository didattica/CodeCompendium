## Introduzione e scenario

È opportuno configurare tutti i dispositivi di rete con almeno un set minimo di comandi di sicurezza comunemente utilizzati come best-practice. Sono compresi i dispositivi degli utenti finali, i server e i dispositivi di rete, quali router e switch.

In questa attività di laboratorio, si configureranno i dispositivi di rete della topologia per accettare sessioni SSH per la gestione remota. Inoltre, si utilizzerà la CLI di IOS per configurare misure di sicurezza di base comuni basate su best-practice. Quindi, si verificheranno le misure di sicurezza per accertarsi della corretta implementazione e del corretto funzionamento.

> [!NOTE]
> I router utilizzati nei laboratori pratici CCNA sono Cisco 4221 con Cisco IOS XE Release 16.9.4 (immagine universalk9). Gli switch utilizzati nelle attività di laboratorio sono Cisco Catalyst 2960s con Cisco IOS Release 15.2(2) (immagine lanbasek9). È possibile utilizzare altri router, switch e versioni di Cisco IOS. A seconda del modello e della versione di Cisco IOS utilizzati, i comandi disponibili e l'output generato potrebbero variare rispetto a quanto visualizzato nei laboratori. Fare riferimento alla Tabella di riepilogo delle interfacce dei router al termine del laboratorio per verificare gli identificatori di interfaccia.

> [!WARNING]
> Accertarsi che i dati presenti sui router e sugli switch siano stati cancellati e che non sia presente nessuna configurazione di avvio. In caso di dubbi, rivolgersi all'istruttore.

## Parte 2: Configurare le misure di sicurezza di base nel router

### Fase 1: Configurare le misure di sicurezza

> [!TIP]
> **Aprire la finestra di configurazione**

a. Criptare tutte le password in chiaro.

b. Configurare il sistema per richiedere una password di minimo 12 caratteri.

c. Modificare le password (privileged exec, console e vty) per soddisfare il nuovo requisito di lunghezza.

   1. Impostare la password privileged EXEC su `$cisco!PRIV*`
   2. Impostare la password della console su `$cisco!!CON*`
   3. Impostare la password della linea vty a `$cisco!!VTY*`

d. Configurare il router per accettare solo connessioni SSH da posizioni remote.

   1. Configurare il nome utente `SSHadmin` con una password crittografata di `55HadM!n2020`
   2. Il nome di dominio del router deve essere impostato su `ccna-lab.com`
   3. Il modulo chiave dovrebbe essere `1024 bit`

e. Impostare le configurazioni di sicurezza e best practice sulle linee console e vty.

   1. Gli utenti devono essere disconnessi dopo `5 minuti` di inattività.
   2. Il router non dovrebbe consentire l'accesso vty per `2 minuti` se `3 tentativi` di accesso non riusciti si verificano entro `1 minuto`.

> [!TIP]
> **Chiudere la finestra di configurazione**

## Parte 3: Configurare le misure di sicurezza

### Fase 1: Verificare che tutte le porte non utilizzate siano disabilitate

Le porte del router sono disabilitate per impostazione predefinita, ma è sempre prudente verificare che tutte le porte non utilizzate si trovino in uno stato di disattivazione amministrativa (`administratively down`). Questo può essere controllato rapidamente utilizzando il comando seguente:

```bash
show ip interface brief
```

Tutte le porte non utilizzate che non si trovano in uno stato di disattivazione amministrativa devono essere disabilitate utilizzando il comando `shutdown` nella modalità di configurazione dell'interfaccia.

## Parte 4: Configurare le misure di sicurezza di base nello switch

### Fase 1: Configurare le misure di sicurezza

> [!TIP]
> **Aprire la finestra di configurazione**

a. Criptare tutte le password in chiaro.

b. Configurare il sistema per richiedere una password di almeno 12 caratteri.

c. Modificare le password (privileged exec, console e vty) per soddisfare il nuovo requisito di lunghezza.

   1. Impostare la password privileged EXEC su `$cisco!PRIV*`
   2. Impostare la password della console su `$cisco!!CON*`
   3. Impostare la password della linea vty a `$cisco!!VTY*`

d. Configurare lo switch per accettare solo connessioni SSH da posizioni remote.

   1. Configurare il nome utente `SSHadmin` con una password crittografata `55HadM!n2020`
   2. Il nome del dominio switch deve essere impostato su `ccna-lab.com`
   3. Il modulo chiave dovrebbe essere `1024 bit`

e. Impostare le configurazioni di sicurezza e best practice sulle linee console e vty.

   1. Gli utenti devono essere disconnessi dopo `5 minuti` di inattività.
   2. L'opzione non dovrebbe consentire l'accesso per `2 minuti` se si verificano `3 tentativi` di accesso non riusciti entro `1 minuto`.

f. Disabilitare tutte le porte inutilizzate.

---

### Fase 2: Verificare che tutte le porte non utilizzate siano disabilitate

Le porte dello switch sono abilitate per impostazione predefinita. Disattivare tutte le porte non in uso nello switch.

a. È possibile verificare lo stato delle porte dello switch utilizzando il comando:

```bash
show ip interface brief
```

b. Utilizzare il comando `interface range` per disattivare più interfacce alla volta.

```bash
interface range
```

c. Verificare che tutte le interfacce inattive siano in stato di disattivazione amministrativa (`administratively down`).

> [!TIP]
> **Chiudere la finestra di configurazione**

| 2811           | Fast Ethernet 0/0 (F0/0)          | Fast Ethernet 0/1 (F0/1)          | Serial 0/0/0 (S0/0/0)   | Serial 0/0/1 (S0/0/1)   |
| 2900           | Gigabit Ethernet 0/0 (G0/0)       | Gigabit Ethernet 0/1 (G0/1)       | Serial 0/0/0 (S0/0/0)   | Serial 0/0/1 (S0/0/1)   |
| 4221           | Gigabit Ethernet 0/0/0 (G0/0/0)   | Gigabit Ethernet 0/0/1 (G0/0/1)   | Serial 0/1/0 (S0/1/0)   | Serial 0/1/1 (S0/1/1)   |

> [!NOTE]
> Per scoprire come è configurato il router, consultare le interfacce per individuare il tipo di router e il numero delle relative interfacce. Non è possibile elencare in una tabella tutte le combinazioni delle configurazioni per ogni classe di router. Questa tabella include gli identificatori delle possibili combinazioni di interfacce Ethernet e seriali nel dispositivo. La tabella non include altri tipi di interfacce che potrebbero appartenere a router specifici. Un esempio potrebbe essere un'interfaccia ISDN BRI. La stringa tra parentesi è l'abbreviazione ufficiale che può essere utilizzata nei comandi di Cisco IOS per rappresentare l'interfaccia.
