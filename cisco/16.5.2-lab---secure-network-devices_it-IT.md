# Introduzione e scenario

È opportuno configurare tutti i dispositivi di rete con almeno un set minimo di comandi di sicurezza comunemente utilizzati come best-practice. Sono compresi i dispositivi degli utenti finali, i server e i dispositivi di rete, quali router e switch.

In questa attività di laboratorio, si configureranno i dispositivi di rete della topologia per accettare sessioni SSH per la gestione remota. Inoltre, si utilizzerà la CLI di IOS per configurare misure di sicurezza di base comuni basate su best-practice.

Quindi, si verificheranno le misure di sicurezza per accertarsi della corretta implementazione e del corretto funzionamento.

> **Nota:**
> I router utilizzati nei laboratori pratici CCNA sono Cisco 4221 con Cisco IOS XE Release 16.9.4 (immagine universalk9).
> Gli switch utilizzati nelle attività di laboratorio sono Cisco Catalyst 2960s con Cisco IOS Release 15.2(2) (immagine lanbasek9).

# Parte 2: Configurare le misure di sicurezza di base nel router

## Fase 1: Configurare le misure di sicurezza

### Aprire la finestra di configurazione

a. Criptare tutte le password in chiaro.

b. Configurare il sistema per richiedere password di minimo 12 caratteri.

c. Modificare le password:

1. Password privileged EXEC: `$cisco!PRIV*`
2. Password console: `$cisco!!CON*`
3. Password VTY: `$cisco!!VTY*`

d. Configurare il router per accettare solo connessioni SSH.

1. Username: `SSHadmin`
2. Password: `55HadM!n2020`
3. Dominio: `ccna-lab.com`
4. Modulo chiave: `1024 bit`

e. Impostare le best practice sulle linee console e VTY.

1. Timeout inattività: `5 minuti`
2. Blocco accesso VTY:

   * 3 tentativi falliti
   * entro 1 minuto
   * blocco per 2 minuti

### Chiudere la finestra di configurazione
