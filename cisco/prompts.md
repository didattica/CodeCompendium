# Prompt Domande

Sto preparando una lezione sul **modulo 16.5.4 di Cisco Packet Tracer**.

Ti fornirò a breve il contenuto originale del modulo.

Una sezione domande finali

Task:

* riporta ogni domanda;
* indica la risposta corretta;
* spiega perché la risposta corretta è giusta;
* spiega anche perché le altre opzioni sono errate o meno appropriate.

Il tuo compito sarà trasformarlo in un documento **Markdown (`.md`) pronto per GitHub**, mantenendo la massima fedeltà possibile al materiale di riferimento.

# Obiettivi principali

* Riprodurre fedelmente la struttura originale del modulo.
* Mantenere invariata la numerazione delle domande
* Organizzare il contenuto usando correttamente le intestazioni Markdown:

  * `#`
  * `##`
  * `###`
  * ecc.
* Produrre un testo chiaro, ordinato e didattico, adatto a una lezione tecnica.
* Non semplificare eccessivamente i concetti tecnici.
* Mantenere coerente la terminologia Cisco presente nel materiale originale.

# Requisiti del contenuto

Se presenti nel materiale originale, includi anche:

* definizioni,
* esempi,
* configurazioni,
* tabelle,
* note,
* punti elenco,
* procedure operative.

# Formato richiesto

Il documento finale deve:

* essere scritto in formato Markdown compatibile con GitHub;
* essere pronto per la pubblicazione diretta su GitHub;
* avere una struttura leggibile e professionale.

# Admonition Block

Quando utile, usa anche blocchi Admonition in stile GitHub.

Esempio:

```md
> [!IMPORTANT]
> ### 💡 Concetto chiave
> "La connettività senza sicurezza è un rischio, non un vantaggio."
```

Puoi utilizzare anche altri tipi di admonition, ad esempio:

* `[!NOTE]`
* `[!TIP]`
* `[!WARNING]`
* `[!CAUTION]`



# Sezione aggiuntiva per il ripasso

Alla fine del documento crea una sezione aggiuntiva contenente:

## Glossario / Ripasso rapido

Realizza una tabella con:

* termini tecnici,
* protocolli,
* concetti,
* acronimi,
* comandi,
* tecnologie Cisco,

citati nelle domande o nelle opzioni di risposta.

La tabella deve essere pensata per un ripasso rapido prima di un esame o laboratorio.

Esempio:

| Termine | Descrizione                                                 |
| ------- | ----------------------------------------------------------- |
| DHCP    | Protocollo per l’assegnazione automatica degli indirizzi IP |
| WPA2    | Protocollo di sicurezza per reti wireless                   |
